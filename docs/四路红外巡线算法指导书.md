四路红外循线算法指导书

适用对象：四轮差速小车 + 四个数字红外循线传感器

⸻

1. 目标与总体原则

本算法的目标不是“某种传感器组合就执行某个固定动作”，而是让小车持续估计黑胶带相对车中心的位置，并根据偏移趋势平滑调整左右轮速度。

系统分成五层：

1. 传感器读取与去抖
2. 黑线位置估计
3. 偏移趋势估计
4. 正常循线与丢线恢复状态机
5. 差速电机控制与速度调度

核心原则：

* 黑线位置不是离散判断，而是连续估计。
* 小车偏得越多，速度越慢、转向越强。
* 短暂读不到黑线时，不应立刻停车。
* 丢线时优先相信“最后一次看到黑线的位置和移动方向”。
* 真正长时间找不到线时，才进入安全停车。

⸻

2. 硬件与信号约定

四个传感器从左到右排列：

左外  左中  右中  右外
LO    LI    RI    RO

统一规定：

检测到黑胶带 = 1
检测到地面/白色 = 0

如果你们模块实际输出相反，只允许在最底层读取 GPIO 时反转一次。后续算法里永远保持“黑色为 1”。

传感器状态组合写作：

[LO, LI, RI, RO]

例如：

1111：四颗都压在黑胶带上
1110：黑胶带偏左
0011：黑胶带偏右
0000：当前没有任何传感器看到黑胶带

⸻

3. 传感器采样与去抖

3.1 建议采样频率

建议至少：

传感器采样：150～250 Hz
电机控制：80～150 Hz

一个较稳妥的默认值：

传感器采样：200 Hz，即每 5 ms 一次
控制循环：100 Hz，即每 10 ms 一次

循线线程必须独立运行，不能被网页请求、摄像头、OCR、语音播报等任务阻塞。

⸻

3.2 去抖规则

红外模块可能受地面反光、胶带边缘、震动影响而短暂跳变。不要一帧变化就立刻信。

每颗传感器建议使用“连续确认”：

连续 2～3 次检测到黑色，才确认进入黑色状态。
连续 2～3 次检测到白色，才确认离开黑色状态。

在 200 Hz 下，连续 3 帧只会增加约 15 ms 延迟，通常可以接受。

不要使用太重的平均滤波。循线需要快速响应，滤波过头会让车已经冲出去了，算法才反应过来。

⸻

4. 黑线位置估计

4.1 给传感器分配横向坐标

默认可使用：

左外 LO = -1.5
左中 LI = -0.5
右中 RI = +0.5
右外 RO = +1.5

负数表示黑线在小车左边，正数表示黑线在小车右边。

当有一个或多个传感器检测到黑胶带时，计算黑线位置：

error_raw =
所有命中传感器坐标之和
÷
命中传感器数量

例如：

1111：
(-1.5 - 0.5 + 0.5 + 1.5) / 4 = 0
1110：
(-1.5 - 0.5 + 0.5) / 3 = -0.5
0011：
(+0.5 + 1.5) / 2 = +1.0
1000：
-1.5
0001：
+1.5

这样得到的 error_raw 表示黑胶带相对小车中心的横向偏差。

⸻

4.2 有效状态与异常状态

对于一条连续黑胶带，正常情况下命中的传感器应该是连续的一段。

有效状态包括：

0001
0010
0011
0100
0110
0111
1000
1100
1110
1111

以下状态通常不是正常连续黑胶带：

0101
1010
1001
1011
1101

这些状态可能来自：

* 反光或抖动；
* 胶带边缘不平；
* 传感器高度不一致；
* 路口、分叉、特殊图案；
* 弯道过急；
* 传感器安装横向不齐。

处理原则：

异常状态不要立刻停车。
短时间内先保持上一帧的估计结果。
如果连续多帧异常，再降低速度并进入恢复逻辑。

⸻

4.3 对误差进行轻量平滑

建议保留两个变量：

error_filtered：平滑后的偏移量
error_rate：偏移变化速度

每次获得有效测量时：

error_filtered =
alpha × error_raw
+
(1 - alpha) × 上一次 error_filtered

推荐初始参数：

alpha = 0.45 ～ 0.70

速度估计：

error_rate =
(error_filtered - previous_error_filtered) / dt

为了避免导数抖动，也可以再做一次轻微平滑。

⸻

5. 正常循线控制：PD 差速控制

5.1 不建议一开始使用积分项

先用 PD：

turn =
Kp × error_filtered
+
Kd × error_rate

其中：

Kp：偏离多少就拉回多少
Kd：抑制左右摇摆，提前刹住转向趋势

初始阶段：

Ki = 0

因为数字红外传感器本身信息不细，积分项很容易在丢线或急弯时积累错误，造成重新找到线后突然猛甩。

⸻

5.2 左右轮差速控制

定义：

error > 0：黑线在小车右侧
error < 0：黑线在小车左侧

则当黑线在右侧时，小车应向右修正，也就是左轮稍快、右轮稍慢。

left_pwm  = base_pwm + turn
right_pwm = base_pwm - turn

第一次接入时必须验证方向：

黑线在右边时，小车是否真的往右修正？
黑线在左边时，小车是否真的往左修正？

如果方向反了，只改一处：

turn = -turn

不要在多个地方分别反转，否则后面很容易乱。

⸻

5.3 转向限幅与变化限幅

必须限制转向输出：

turn = clamp(turn, -turn_max, +turn_max)

还应限制每个控制周期内 PWM 的变化量：

每 10 ms 内，PWM 不允许跳变过大。

这样可以避免：

* 电机突然反向；
* 小车机械震动；
* 在边缘状态来回闪烁时疯狂抖动；
* 因瞬间误判造成甩尾。

⸻

6. 速度调度：偏得越多，跑得越慢

循线速度不要固定。

建议根据偏差和偏移变化速度自动降速：

base_pwm =
normal_pwm
-
A × abs(error_filtered)
-
B × abs(error_rate)

然后限制范围：

track_min_pwm ≤ base_pwm ≤ normal_pwm

建议逻辑：

接近中心、变化平稳：正常速度
偏离中等：中等速度
只剩外侧单颗命中：低速
刚丢线或恢复中：更低速

可以先用状态表实现：

1111 / 0110：正常速度
1110 / 0111 / 1100 / 0011：中速
1000 / 0001 / 0100 / 0010：低速
0000：恢复速度

速度比精确 PWM 更重要。大多数“算法跟不上”的问题，本质上是车跑得超过传感器与电机响应能力。

⸻

7. 丢线恢复状态机

不要把 0000 直接视为“停车”。

0000 的意思只是：

当前这一小段时间内，没有传感器看到黑胶带。

完整状态机如下：

TRACK
BRIDGE
BIASED_SEARCH
SWEEP_SEARCH
REACQUIRE
FAILSAFE

⸻

7.1 TRACK：正常循线

进入条件：

检测到有效、连续的黑线状态。

在这个状态中：

* 计算 error_raw；
* 更新 error_filtered；
* 更新 error_rate；
* 执行 PD 控制；
* 记录最后一次可靠方向。

必须保存：

last_valid_error
last_valid_error_rate
last_turn_direction
last_seen_time
last_valid_mask

其中：

last_turn_direction：
-1 表示最后黑线主要在左侧
+1 表示最后黑线主要在右侧

⸻

7.2 BRIDGE：短暂失线桥接

当状态从“有黑线”变成 0000 时，不要停车，先进入 BRIDGE。

适用情况：

* 胶带边缘瞬间漏检；
* 传感器抖动；
* 小车轻微晃动；
* 急弯中短暂离开传感器区域；
* 传感器信号切换造成的空窗。

BRIDGE 的动作：

降低速度。
保持上一刻的转向方向。
继续使用最后的 error_filtered 和 error_rate 预测。

推荐默认值：

bridge_time = 60 ～ 120 ms
bridge_speed = 正常速度的 30%～50%

预测逻辑：

predicted_error =
last_error_filtered
+
last_error_rate × dt

但预测不能无限延续，应逐渐衰减：

last_error_rate 每个周期乘一个小于 1 的衰减系数。

例如：

error_rate *= 0.85

这样不会因为旧趋势一直向一个方向越想越偏。

⸻

7.3 BIASED_SEARCH：朝最后方向找线

如果 BRIDGE 超时后仍然是 0000，进入 BIASED_SEARCH。

方向优先级：

1. last_valid_error 的正负
2. last_valid_error_rate 的正负
3. last_turn_direction
4. 上一次转向输出方向

例如：

最后黑线在左边：
低速向左走弧线寻找。
最后黑线在右边：
低速向右走弧线寻找。

推荐动作：

前进速度很低。
保持一个固定但不极端的差速转向。
不要一开始原地猛转。

推荐默认范围：

search_speed = 正常速度的 20%～35%
biased_search_time = 150 ～ 350 ms

原因是多数情况下，小车只是偏离了轨道，黑线仍在前方一侧；低速弧线比原地乱转更容易重新压到线。

⸻

7.4 SWEEP_SEARCH：左右扫描找线

如果朝最后方向找了一段时间仍然找不到，进入 SWEEP_SEARCH。

动作逻辑：

先向最后已知方向扫一小段。
再反向扫回来。
之后每次扫动时间略微增加。

示例：

左 80 ms
右 120 ms
左 160 ms
右 220 ms

注意：

* 全程低速；
* 总搜索时间必须有限；
* 扫描范围不要无限扩大；
* 若有障碍物模块，搜索时必须仍受避障系统约束。

推荐：

sweep_total_timeout = 0.8 ～ 1.5 秒

⸻

7.5 REACQUIRE：重新找到线后的平滑接管

在恢复状态下重新检测到有效黑线时，不要立即恢复满速。

进入条件建议：

连续 2～3 个采样周期检测到有效连续状态。

进入后：

清空或大幅衰减旧的 error_rate。
将 error_filtered 平滑拉向新的 error_raw。
速度从恢复速度逐步回升。

建议：

reacquire_ramp_time = 100 ～ 250 ms

这样可避免：

丢线向左找
→ 找到线后误差突然变成右
→ 控制器全力反向
→ 又把车甩过线

⸻

7.6 FAILSAFE：真正丢线后的安全停止

以下情况进入 FAILSAFE：

总恢复时间超时；
搜索范围已经走完；
连续长时间都是 0000；
传感器状态持续异常；
电机输出正常但位置无法恢复。

动作：

左右轮平滑减速至 0。
停止循线。
网页或终端显示“循线丢失”。
记录最近 1～2 秒日志。

不要让小车无限找线。无限乱跑比停下来更危险，也更难调试。

⸻

8. 主控制流程伪代码

每 5 ms：
    raw = 读取四颗传感器
    filtered_bits = 去抖后的传感器状态
    mask = 组合四位状态
每 10 ms：
    if mask 是有效连续状态：
        error_raw = 命中传感器坐标平均值
        更新 error_filtered
        更新 error_rate
        last_valid_error = error_filtered
        last_valid_error_rate = error_rate
        last_turn_direction = sign(error_filtered 或 turn)
        if 当前状态不是 TRACK：
            进入 REACQUIRE
    else if mask 是 0000：
        根据当前状态执行：
            TRACK → BRIDGE
            BRIDGE → BIASED_SEARCH
            BIASED_SEARCH → SWEEP_SEARCH
            SWEEP_SEARCH → FAILSAFE
    else:
        视为短时异常状态
        保持上一帧估计
        降低速度
        若异常持续过久，进入恢复逻辑
    根据状态决定：
        TRACK：
            使用 PD 控制
            根据误差动态调速
        BRIDGE：
            使用预测误差
            低速保持原转向趋势
        BIASED_SEARCH：
            低速朝最后已知方向转弯搜索
        SWEEP_SEARCH：
            按预设时序左右扫线
        REACQUIRE：
            低速重新接入 PD
            逐渐恢复速度
        FAILSAFE：
            平滑停车

⸻

9. 推荐初始参数

以下不是最终答案，只是第一轮可跑的起点。

sensor_sample_hz = 200
control_hz = 100
debounce_frames = 2 或 3
error_filter_alpha = 0.55
error_rate_filter_alpha = 0.35
Ki = 0
bridge_time = 80 ms
biased_search_time = 250 ms
sweep_total_timeout = 1000 ms
reacquire_ramp_time = 150 ms
normal_speed = 你们当前稳定直行速度的 70%～80%
medium_speed = normal_speed 的 70%
slow_speed = normal_speed 的 40%～50%
search_speed = normal_speed 的 25%～35%
turn_max = 电机允许的安全差速上限

Kp、Kd 不建议直接照抄固定值，因为你们的 PWM 范围、电机减速比、车重、轮胎、地面摩擦都不同。

⸻

10. 参数整定顺序

第一步：先校准电机直行

关闭循线，只让左右轮相同 PWM 前进。

目标：

在平整地面上，车不要明显偏左或偏右。

如果有机械偏差，可加入基础补偿：

left_base_pwm
right_base_pwm

不要让循线算法长期替电机偏差擦屁股。

⸻

第二步：只调 Kp

先设：

Kd = 0
Ki = 0
速度调低

逐渐增大 Kp，直到小车开始能拉回黑线。

现象判断：

一直回不来：Kp 太小
明显左右摆动：Kp 太大

找到“刚开始摆动”的临界点后，将 Kp 降低约 15%～25%。

⸻

第三步：再加 Kd

逐渐增加 Kd，直到左右摆动明显减轻。

现象判断：

摆动大：Kd 太小
反应迟钝、转不过弯：Kd 太大

Kd 的作用是压住惯性，不是让车变慢。

⸻

第四步：逐步提高速度

每次只提高一点速度，并重复跑完整赛道。

一旦出现：

弯道冲出线
不断蛇形摆动
外侧传感器频繁触发
丢线恢复次数明显变多

说明速度已经高于当前控制能力。

先降速，不要急着把 Kp、Kd 调得特别激进。

⸻

11. 测试流程

11.1 静态横移测试

将小车架起，或固定传感器条。

让黑胶带在传感器下方缓慢横向移动，记录状态变化。

理想现象：

1000
1100
1110
1111
0111
0011
0001

反向移动时顺序反过来。

允许中间出现：

0100
0110
0010

但不应该在正常横移中频繁出现：

0000
0101
1010
1001

如果出现，优先检查：

* 传感器高度；
* 发射/接收灵敏度；
* 胶带是否反光；
* 传感器是否横向对齐；
* 相邻传感器间距；
* 地面颜色与材质。

⸻

11.2 慢速闭环测试

先用低速跑直线。

观察日志：

传感器 mask
error_raw
error_filtered
error_rate
状态机状态
left_pwm
right_pwm

目标：

error 不应频繁正负大幅跳变。
turn 不应剧烈跳变。
小车应平稳回到中心。

⸻

11.3 S 弯与急弯测试

逐渐增加弯道难度。

重点看：

外侧单颗传感器触发时是否自动减速；
是否能在外侧检测到线后及时拉回；
是否出现来回冲线。

⸻

11.4 人为偏离测试

将小车轻轻推到黑线一侧，再放开。

验证：

是否优先向最后看到黑线的一侧找回；
是否会立即停车；
重新找到线后是否平滑恢复；
是否出现猛甩。

⸻

11.5 真丢线测试

把车移到完全无胶带区域。

验证：

BRIDGE
→ BIASED_SEARCH
→ SWEEP_SEARCH
→ FAILSAFE

整个过程必须可预测，不应无限乱跑。

⸻

12. 日志与调试要求

每次循线测试最好记录一条简洁日志：

时间
mask
状态机状态
error_raw
error_filtered
error_rate
left_pwm
right_pwm

例如：

12.340  1111  TRACK           0.00   0.01   0.02   62   61
12.350  1110  TRACK          -0.50  -0.28  -2.90   52   70
12.360  0000  BRIDGE         -0.42  -0.36  -2.10   45   66
12.430  1100  REACQUIRE      -1.00  -0.72  -1.40   36   72

不要只看“小车跑得不对”。要先看它在错误发生前的传感器序列、状态机状态和 PWM 输出，才能知道问题在：

* 传感器；
* 参数；
* 电机；
* 算法；
* 赛道；
* 机械安装。

13. 路口、横线与特殊轨迹

现在宽胶带居中时可能出现 1111，因此：

不能把 1111 单独当作“路口”。

若赛道有十字、T 字、横线、直角转弯，需要额外做“轨迹层逻辑”。

建议原则：

基础循线层只负责跟住当前黑胶带。
路口判断必须结合持续时间、赛道任务、前后状态和预期路径。

例如：

1111 持续极短时间：普通居中循线
1111 持续明显更久：可能是横线或路口
前方任务要求右转：路口出现后进入右转状态

不要把基础 PD 循线和“指定路口转向”混成一堆 if-else。

LineSensorReader
    读取 GPIO
    极性统一
    去抖
    输出四位 mask
LineEstimator
    判断 mask 是否有效
    计算 error_raw
    平滑 error_filtered
    计算 error_rate
LineStateMachine
    TRACK
    BRIDGE
    BIASED_SEARCH
    SWEEP_SEARCH
    REACQUIRE
    FAILSAFE
LineController
    PD 控制
    速度调度
    输出左右轮目标 PWM

    PWM 限幅
    PWM 平滑
    左右电机执行

    保存 mask、误差、状态、PWM

mask 输出

BIASED_SEARCH

路口

与避障、OCR、AprilTag 等模块协同

16. 最终验收标准

基础循线完成后，应达到：

正常直线时不蛇形摆动；
普通弯道不轻易丢线；
偏到一侧后可平滑拉回；
短暂 0000 不立即停车；
重新压到胶带后不猛甩；
真正丢线后能有限搜索并安全停止；
所有异常都能从日志中复盘。

这套算法的核心不是某个神奇参数，而是把“传感器当前状态、过去趋势、恢复逻辑、电机响应”连成一条完整链路。硬件连续覆盖已经解决了最危险的盲区，剩下最重要的是：低速调稳、记录日志、按顺序调参，而不是一开始就追求最快速度。
