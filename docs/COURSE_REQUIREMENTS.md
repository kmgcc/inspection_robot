# 课程交付要点摘要

这份摘要来自本地的课程模板和注意事项，目的是提醒后续开发时不要只顾写代码。

## 报告必须讲清楚

- 应用场景：为什么要做移动式物品巡检。
- 业务逻辑：小车、视觉标签、异常上报、人工确认之间的数据流。
- 主要功能：巡检、识别、异常判定、声光告警、网页看板、人工确认。
- 特色亮点：AprilTag 让物品身份可验证，网页看板让过程可追踪，人工确认让异常处理形成闭环。
- 硬件结构：树莓派、小车底盘、摄像头、超声波、巡线、蜂鸣器、RGB。
- 软件设计：状态机、标签映射表、Web API、后续 ROS2 接入。
- 调试说明：连接问题、摄像头问题、标签识别不稳、端口占用、电量不足、巡线偏差。
- 程序清单：最终提交时要附核心代码和注释。

## 答辩必须能演示

```text
打开网页看板 -> 开始巡检 -> 识别标签 -> 异常上报 -> 声光告警 -> 人工确认 -> 事件关闭
```

演示时不必追求功能堆得多，关键是每一步都能解释、能复现、能从网页上看见结果。

## 现场注意

- 小车电量会直接影响巡线和转向，验收前一晚要充满电。
- 不运行时尽量充电，但充满后及时拔掉。
- 黑线建议用黑色胶带，路线不要太复杂。
- 舵机和巡线模块如果不准，先检查中位和灵敏度，不要先怀疑程序。
- 资料和代码要随时备份，树莓派系统重刷后本地文件会丢。
