# 货架通道巡逻演示运行手册

**依据：** `docs/REAL_REQUIREMENTS.md`。本文不再使用“固定栅格地图规划路径”的演示叙事。

## 一、场地布置

1. 白色地面或浅色地面。
2. A 列货架按行驶方向摆放为 `A1`、`A2`、`A3`、`A4`。
3. B 列货架编号按现场确认后摆放。
4. 每个货架贴货架 AprilTag，文字编号可放在 AprilTag 上方作 OCR 辅助。
5. 每个物品尽量提供 AprilTag、文字和图形标识；颜色有则保留，没有不影响演示。
6. 货架列尽头铺设横向黑胶带，使四路传感器能同时检测到黑色。
7. 每列尽头用“工字形”黑胶带形成端点触发，帮助小车在列间往返循环。

## 二、真车运行前检查

1. 官方 App 主程序已禁用或运行前已清理。
2. Raspberry Pi Wi-Fi 已连接。
3. SSH 可访问。
4. VNC 可访问。
5. 网页看板可访问。
6. 页面显示 `RUN_MODE=robot` 或等价真车模式。
7. 急停按钮可见。
8. 云台初始化按钮可用。

## 三、真车演示流程

1. 部署代码：

```bash
scripts/deploy_to_car.sh
```

2. 启动真车模式：

```bash
RUN_MODE=robot scripts/run_on_car.sh
```

3. 浏览器打开：

```text
http://小车IP:5000
```

4. 点击“云台初始化”，确认摄像头朝向货架侧面。
5. 点击“前进”和“停止”短测手动控制。
6. 点击“开始巡逻”，小车应慢速向前。
7. 第一轮经过货架时只记录观察，不报缺货。
8. 到列端四路全黑，小车顺时针原地转 90 度并继续。
9. 第二轮开始，制造一个缺货场景。
10. 看板显示 `missing_item`，红灯和语音警报触发。
11. 放置障碍物，小车停止等待 6 秒。
12. 如果移走障碍，小车恢复；如果不移走，小车进入绕行。
13. 点击“确认处理”，异常事件变为已确认。
14. 导出 CSV。

## 四、软件兜底流程

本地启动：

```bash
scripts/run_local.sh
```

打开：

```text
http://127.0.0.1:5050
```

软件兜底应演示：

1. 启动巡逻。
2. 云台初始化事件。
3. 第一轮识别 A1-A4，不报缺货。
4. 列端四路全黑转向事件。
5. 第二轮扫描并产生缺货事件。
6. 红灯和语音警报事件。
7. 障碍等待 6 秒。
8. 绕行步骤。
9. 人工确认。
10. 导出 CSV。

## 五、讲解顺序

| 时间 | 动作 | 讲解点 |
|---|---|---|
| 0:00-0:20 | 打开网页 | 固定货架通道，不做 SLAM，不是循线 |
| 0:20-0:45 | 启动巡逻 | 小车慢速正向行驶，摄像头侧向看货架 |
| 0:45-1:10 | 货架识别 | 货架 AprilTag，物品 AprilTag/文字/图形，颜色可选 |
| 1:10-1:30 | 列端转向 | 四路全黑触发顺时针 90 度 |
| 1:30-1:55 | 第二轮缺货 | 第一轮跳过，第二轮开始判断缺货 |
| 1:55-2:20 | 障碍处理 | 停车等待 6 秒，仍有障碍则绕行 |
| 2:20-2:40 | 确认和导出 | 人工确认、CSV 和事件闭环 |

## 六、硬件失败兜底

1. 底盘失败：使用软件兜底演示状态、事件、轮次和异常。
2. AprilTag 不稳：展示录屏或截图，再用模拟事件说明规则。
3. 音频失败：展示网页音频事件和灯光提示。
4. 网络失败：使用 SSH 查看日志，或用本地录屏。
5. VNC 不可用：保留 SSH 和网页作为主要演示路径。
