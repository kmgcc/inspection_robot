# 0.1 基准计划：共享契约、配置格式与资料依据

**负责人：** 基准协调者  
**状态：** 旧版 0.1 最小契约已完成；新版仓库巡逻目标需要按本文做一次对齐修订。  
**目标：** 让三个队友可以并行开发，而不在字段、配置、事件名称和资料依据上互相猜。

本计划不直接做硬件、不写路径规划算法、不改看板样式。它只管五件事：`/api/status` 返回什么，事件对象长什么样，地图/货架/标签配置怎么写，AprilTag 与多模态识别证据如何落表，三份子计划该依赖哪些稳定接口。这个基准一旦定下，1.1 可以接小车硬件，2.1 可以写核心算法，3.1 可以做看板和演示。

## Task 0.1.2：定义配置文件格式

**Files:**

- Create: `config/warehouse_map.json`
- Create: `config/shelf_manifest.json`
- Modify: `config/tag_map.json`
- Modify: `docs/api_contract.md`

**Step 1：仓库地图**

`warehouse_map.json` 用固定小栅格表达演示场地：

```json
{
  "grid_size": [8, 6],
  "start": [0, 0],
  "home": [0, 0],
  "forbidden_cells": [[2, 2], [2, 3]],
  "shelf_points": {
    "A1": {"scan_pose": [3, 1, "E"], "safe_side": "W"},
    "A2": {"scan_pose": [5, 1, "E"], "safe_side": "W"}
  }
}
```

`safe_side` 用来告诉避障逻辑哪边更安全。比如摄像头面向货架时，绕障应优先往货架相反方向移动，避免小车向货架挤压。

**Step 2：货架清单**

`shelf_manifest.json` 表达每个货架应有物品：

```json
{
  "A1": {"expected_items": ["item_01", "item_02", "item_03"]},
  "A2": {"expected_items": ["item_04", "item_05"]}
}
```

**Step 3：标签字典**

`tag_map.json` 继续保留，但扩展 `kind`：

```json
{
  "118": {
    "name": "A1",
    "kind": "shelf",
    "shelf_id": "A1",
    "marker_family": "TAG36H11",
    "ocr_label": "A1"
  },
  "46": {
    "name": "Red Bottle",
    "kind": "item",
    "item_id": "item_46",
    "expected_shelf": "A1",
    "marker_family": "TAG36H11",
    "expected_color": "RED",
    "expected_ocr": "ITEM-46",
    "expected_image_class": "BOTTLE",
    "priority": 1
  }
}
```

AprilTag、二维码、文字识别都可以统一落成 `tag_id` 或识别证据。核心系统以 AprilTag TAG36H11 的数字 ID 作为主身份来源，再用 OCR、颜色和图像分类做补充判断。

**Step 4：ID 范围与打印素材**

统一采用以下 ID 范围：

```text
1-50      物品标签
101-120   货架标签
201-220   定位点预留
301-320   禁区/特殊点预留
```

同一个 ID 不能同时定义为货架和物品。货架标签版式固定为“上方大号货架号、下方 AprilTag、底部数字脚注”；物品标签版式固定为“颜色块、图像、文字、AprilTag、底部数字脚注”。根目录 `打印素材_AprilTag/manifest.csv` 可作为配置初稿来源。

**Verify:**

```bash
python3 -m json.tool config/warehouse_map.json >/dev/null
python3 -m json.tool config/shelf_manifest.json >/dev/null
python3 -m json.tool config/tag_map.json >/dev/null
```

Expected: 三个 JSON 都能解析。
