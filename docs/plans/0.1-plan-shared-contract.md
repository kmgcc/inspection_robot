# 0.1 Plan Implementation Plan

**状态：** 已完成，契约文件见 docs/api_contract.md。

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Define and implement the smallest shared contract that lets the robot-side plan, core plan, and dashboard plan proceed in parallel.

**Architecture:** This is the only prerequisite plan. It freezes `/api/status`, event fields, status enums, and the small set of `InspectionStore` methods used by the robot runtime and dashboard. It deliberately avoids full rule-engine work, persistence, and dashboard redesign so it can finish quickly.

**Tech Stack:** Python 3, Flask test client, stdlib `unittest`, Markdown contract document.

---

## 为什么先做 0.1

1.1 和 3.1 的真正交界点不是完整业务规则，而是这几件事：

- 小车识别到标签后，调用什么方法把 tag ID 交给系统。
- 小车发现障碍后，调用什么方法告诉系统当前 blocked / clear。
- 看板从 `/api/status` 读哪些字段。
- 看板如何判断“待确认异常”和“最新事件”。
- 事件对象里哪些字段永远存在。

这些定完以后：

- 1.1 可以继续做硬件、AprilTag、巡线、告警和 runtime。
- 2.1 可以继续做异常规则、状态机、持久化和测试。
- 3.1 可以继续做看板 UI、演示流程和部署文档。

## 当前仓库起点

- `src/inspection_robot/state.py` 已有 `InspectionStore`，但状态字段仍偏模拟版。
- `src/inspection_robot/web.py` 已有 `/api/status`、`/api/simulate/tag/<tag_id>`、`/api/confirm`、`/api/export.csv`。
- `src/inspection_robot/static/dashboard.js` 当前依赖 `task_status`、`robot_status`、`current_tag`、`current_item`、`current_zone`、`events`。
- 当前还没有 `docs/api_contract.md` 和 `tests/`。

## Task 0.1.1: 写共享契约文档

**Files:**

- Create: `docs/api_contract.md`

**Step 1: 写 `/api/status` 最小结构**

`docs/api_contract.md` 必须规定 `/api/status` 至少返回：

```json
{
  "run_id": "local-001",
  "task_status": "PATROL",
  "robot_status": "巡检中",
  "current_zone": "A区",
  "current_tag": "1",
  "current_item": "Apple",
  "last_message": "标签 1 识别正常。",
  "obstacle": {
    "distance_mm": 320,
    "blocked": false
  },
  "alarm": {
    "level": "normal",
    "message": "正常"
  },
  "zones": [],
  "events": []
}
```

**Step 2: 写内部状态枚举**

状态内部统一用英文：

```text
IDLE
PATROL
TAG_DETECTED
NORMAL_LOGGED
ABNORMAL_ALARM
WAIT_CONFIRM
CONFIRMED
OBSTACLE_WAIT
FINISHED
STOPPED
```

页面显示中文，由 3.1 的 `dashboard.js` 做映射。

**Step 3: 写 event 最小结构**

每个事件必须包含：

```json
{
  "id": "evt-0001",
  "time": "2026-07-02T10:00:00",
  "type": "wrong_zone",
  "tag_id": "4",
  "item": "Bottle",
  "zone": "A区",
  "expected_zone": "B区",
  "priority": 2,
  "status": "waiting_confirm",
  "message": "Bottle 出现在 A区，期望 B区。"
}
```

**Step 4: 写 event type 和 status**

先固定这些类型：

```text
normal_tag
unknown_tag
wrong_zone
missing_tag
duplicate_tag
obstacle_wait
obstacle_clear
manual_confirm
system
```

事件状态先固定：

```text
normal
waiting_confirm
confirmed
info
```

**Step 5: 写 store 方法契约**

文档中写明 `InspectionStore` 必须提供：

```python
def record_tag(self, tag_id: str, observed_zone: str | None = None, source: str = "simulate") -> None: ...
def record_obstacle(self, distance_mm: int | None, blocked: bool) -> None: ...
def record_robot_status(self, status: str, message: str | None = None) -> None: ...
def confirm(self, event_id: str | None = None) -> bool: ...
def export_events_csv(self) -> str: ...
```

**Step 6: Verify**

```bash
test -f docs/api_contract.md
rg -n "record_tag|record_obstacle|record_robot_status|/api/status|waiting_confirm" docs/api_contract.md
```

Expected: 契约文档存在，并包含 1.1 和 3.1 需要的所有字段和方法。

## Task 0.1.3: 补齐 robot 和 dashboard 共用方法

**Files:**

- Modify: `src/inspection_robot/state.py`

**Step 1: 增加 `record_tag`**

`record_tag()` 先可以委托当前 `handle_tag()`：

```python
def record_tag(self, tag_id: str, observed_zone: str | None = None, source: str = "simulate") -> None:
    self.handle_tag(tag_id)
```

后续 2.1 再把它替换成完整规则引擎。

**Step 2: 增加 `record_obstacle`**

行为：

- `blocked=True` 时，`task_status = "OBSTACLE_WAIT"`。
- 写入 `obstacle.distance_mm` 和 `obstacle.blocked`。
- 增加或更新一条 `obstacle_wait` / `obstacle_clear` 事件。

**Step 3: 增加 `record_robot_status`**

行为：

- 只更新 `task_status` / `robot_status` / `last_message`。
- 不做业务规则判断。

**Step 4: 保持旧模拟接口可用**

`web.py` 里的 `/api/simulate/tag/<tag_id>` 可以继续调用 `handle_tag()`，也可以改为调用 `record_tag(tag_id)`。

**Step 5: Verify**

```bash
py -3 -m py_compile app.py src/inspection_robot/*.py
```

Expected: 编译通过；旧模拟入口仍可运行。
