# 0.1 基准计划：共享契约与真实需求对齐

**前置依据：** `docs/REAL_REQUIREMENTS.md`  
**目标：** 统一接口、事件、配置和状态字段，让硬件、核心逻辑和网页都按真实巡逻需求开发。

## 当前主线

0.1 不再围绕固定栅格地图、A* 路径规划或旧区域标签接口设计。新的基准是：

1. 货架通道循环巡逻。
2. 四路全黑列端触发顺时针 90 度转弯。
3. 第一轮跳过缺货检测。
4. 第二轮开始检测缺货等异常。
5. 货架 AprilTag 识别。
6. 物品 AprilTag、文字、图形识别，颜色可选。
7. 运行中生成拓扑或轨迹，不在网页初始展示假的固定地图。
8. 网页必须区分 `simulate` 和 `robot`。

## Task 0.1.1：冻结新版 API 契约

**Files:**

- Modify: `docs/api_contract.md`
- Modify: `tests/test_contract.py`

契约必须包含：

1. `run_mode`。
2. `hardware_connected`。
3. `patrol_cycle`。
4. `skip_shortage_detection`。
5. `boundary`。
6. `topology`。
7. `audio`。
8. `gimbal`。
9. `alarm`。
10. `scan.detections`。

旧字段可以兼容，但不能继续作为新主链路的设计中心。

## Task 0.1.2：定义配置和拓扑格式

**Files:**

- Modify: `config/shelf_manifest.json`
- Modify: `config/tag_map.json`
- Optional: keep `config/warehouse_map.json` only for software兜底或旧测试
- Modify: `docs/api_contract.md`

要求：

1. 货架清单至少能表达 `A1`、`A2`、`A3`、`A4`。
2. B 列编号待确认后补入。
3. 标签字典区分 `shelf` 和 `item`。
4. 物品字段支持 AprilTag、文字、图形、颜色。
5. 颜色可选，缺失不报错。
6. 动态拓扑记录货架节点、列端转向点、边界点、异常标记和当前节点。
