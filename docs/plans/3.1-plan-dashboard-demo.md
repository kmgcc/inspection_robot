# 3.1 队友计划：网页看板、真车控制与演示兜底

**前置依据：** `docs/REAL_REQUIREMENTS.md`、`docs/api_contract.md`  
**目标：** 网页必须真实反映当前运行模式和硬件状态，提供可用的真车控制、动态拓扑显示、异常确认、日志导出和软件兜底。

## 基本原则

1. 初始不显示假的固定栅格地图。
2. 页面必须显示 `simulate` 或 `robot`。
3. 页面必须显示是否连接真实硬件。
4. 真车模式下，开始按钮要能让小车运动。
5. 手动控制要能用于现场调试。
6. 急停必须始终明显可用。

## Task 3.1.1：第一屏状态区

**Files:**

- Modify: `src/inspection_robot/templates/dashboard.html`
- Modify: `src/inspection_robot/static/dashboard.js`
- Modify: `src/inspection_robot/static/styles.css`

第一屏显示：

1. 运行模式：`simulate` / `robot`。
2. 硬件连接状态。
3. 当前状态。
4. 当前轮次。
5. 是否第一轮跳过缺货检测。
6. 当前货架。
7. 最近货架 AprilTag。
8. 最近物品。
9. 最近 OCR、图形、颜色证据。
10. 障碍距离。
11. 黑胶带/列端状态。
12. 音频和灯光状态。
13. 待确认异常数量。

## Task 3.1.2：动态拓扑视图

不要默认展示固定网格地图。

页面逻辑：

1. 初始显示“尚未生成巡检地图”。
2. 识别到货架后添加货架节点。
3. 识别到列端四路全黑后添加转向点。
4. 识别到异常后在货架节点上标记。
5. 可使用拓扑图、路线图、货架列表加转向点示意或时间线。
6. 不要求真实比例，不宣称 SLAM。

## Task 3.1.3：真车控制区

按钮：

1. 开始巡逻。
2. 急停。
3. 停止。
4. 前进。
5. 后退。
6. 左移。
7. 右移。
8. 左转 90 度。
9. 右转 90 度。
10. 云台初始化。
11. 货架提示音测试。
12. 物品提示音测试。
13. 异常语音测试。

要求：

1. `simulate` 模式下按钮应明确显示不可控制真车的原因。
2. `robot` 模式下按钮应发送真实控制请求。
3. 点击失败时给出可操作错误说明，例如“当前服务不是 robot mode，请用 RUN_MODE=robot 启动”。
4. 急停不能被隐藏在普通按钮中。

## Task 3.1.4：事件与异常

事件表至少展示：

1. 时间。
2. 轮次。
3. 类型。
4. 货架。
5. 物品。
6. 标签 ID。
7. OCR。
8. 图形。
9. 颜色。
10. 状态。
11. 来源。
12. 说明。

待确认异常可人工确认。

## Task 3.1.5：软件兜底

软件兜底不再演示“固定地图规划路径”，而演示真实主链路的状态变化：

1. 启动巡逻。
2. 云台初始化。
3. 第一轮按 `A1 -> A2 -> A3 -> A4 -> B4 -> B3 -> B2 -> B1` 识别货架但不报缺货。
4. 四路全黑列端触发顺时针转向。
5. 第二轮扫描并发现缺货。
6. 触发红灯和语音警报事件。
7. 模拟障碍等待 6 秒。
8. 模拟绕行步骤。
9. 人工确认异常。
10. 导出日志。

## Task 3.1.6：部署与运行模式

**Files:**

- Modify: `scripts/run_on_car.sh`
- Modify: `scripts/install_car_autostart.sh`
- Modify: `README.md`

3. 页面可见当前模式。
4. 手动控制依赖 robot mode。
5. 开机自启不能破坏 Wi-Fi、SSH、VNC。
6. 启动前清理官方 App 进程。

## Done Gate

1. 页面不再显示假的固定地图。
2. 页面明确显示运行模式和硬件状态。
3. 开始按钮在 robot mode 下能启动小车。
4. 手动控制按钮在 robot mode 下可用。
5. simulate mode 下错误说明清楚。
6. 动态拓扑可显示货架、列端、异常。
7. 软件兜底演示真实巡逻主链路。
8. 日志可导出。
