# 3.1 队友计划：看板、模拟演示、部署与答辩证据

**负责人：** 队友 3  
**前置依赖：** `docs/api_contract.md`；可先按缺省字段做前端兜底  
**目标：** 把网页看板和演示流程改成新版仓库巡逻系统：展示固定地图、禁区、路径、当前目标、货架扫描、多模态识别证据、异常事件、人工确认处理和日志导出。

本计划不写硬件驱动，也不复制 2.1 的规则。看板只消费 `/api/status` 和事件列表；模拟按钮只用于演示和兜底，不能成为核心业务的第二套实现。

## 当前仓库起点

- `src/inspection_robot/templates/dashboard.html` 是单页看板，但仍是旧的物品/分区视角。
- `src/inspection_robot/static/dashboard.js` 每秒轮询 `/api/status`，但对新版字段还没有兜底。
- `src/inspection_robot/static/styles.css` 是基础样式。
- `scripts/run_local.sh`、`scripts/run_on_car.sh` 已能启动服务。
- README 仍需与新版目标对齐。

## Task 3.1.1：新版看板信息架构

**Files:**

- Modify: `src/inspection_robot/templates/dashboard.html`
- Modify: `src/inspection_robot/static/styles.css`
- Modify: `src/inspection_robot/static/dashboard.js`

**Step 1：第一屏状态区**

第一屏必须显示：

- 当前阶段。
- 当前目标点。
- 当前货架。
- 当前标签/物品。
- 货架 OCR 结果。
- 物品颜色、文字和图像类别证据。
- 前方障碍距离和是否阻塞。
- 告警等级。
- 当前路径状态。
- 待确认异常数量。

旧字段缺失或新字段未实现时显示 `-`，不要让页面出现 `undefined`。

**Step 2：地图与路径区**

用简单网格或表格展示固定地图：

- 起点。
- 货架点 A1/A2/B1/B2。
- 黑胶带禁区或配置禁区。
- 当前小车估计位置。
- 已规划路径。

第一版可以不用 canvas，HTML grid 即可。地图要服务演示，不必做复杂交互。

**Step 3：货架卡片**

把旧的 A/B/C/D 分区卡片改为货架卡片：

```text
A1：未巡检 / 扫描中 / 正常 / 异常 / 待确认
A2：未巡检 / 扫描中 / 正常 / 异常 / 待确认
B1：...
B2：...
```

每张卡片显示异常数量和最近识别物品。

货架卡片还要显示主识别和补充识别：

```text
A1
AprilTag: 118
OCR: A1
状态: 扫描中
异常: 0
```

**Step 4：事件表**

事件表至少展示：

- 时间
- 类型
- 货架
- 标签
- 物品
- 期望货架
- 状态
- 来源
- 说明
- OCR/颜色/图像类别，若事件中有这些字段。

旧事件缺少 `shelf_id` 时，可以用 `expected_zone` 或 `zone` 兜底。

**Step 5：操作按钮**

保留并改名：

- 开始巡逻
- 停止
- 重置
- 模拟规划路径
- 模拟障碍等待
- 模拟障碍解除
- 模拟扫描 A1 正常
- 模拟扫描 A2 异常
- 模拟识别证据冲突
- 确认处理
- 导出日志

旧版确认按钮统一改成“确认处理”或“人工复核确认”。

**Verify:**

```bash
scripts/run_local.sh
```

Expected: `http://127.0.0.1:5050` 页面可打开；旧 API 字段下不报错；移动端宽度文字不重叠。

## Task 3.1.2：按契约接入新版字段

**Files:**

- Modify: `src/inspection_robot/static/dashboard.js`
- Modify: `tests/test_web_api.py` 或 `tests/test_contract.py`

**Step 1：集中状态映射**

在 `dashboard.js` 中建立：

```javascript
const STATUS_LABELS = {
  IDLE: "待命",
  PLANNING: "规划中",
  PLAN_READY: "路径就绪",
  MOVING: "移动中",
  ALIGNING_SHELF: "对准货架",
  SCANNING_SHELF: "扫描货架",
  ANALYZING: "分析中",
  NORMAL_LOGGED: "正常已记录",
  ABNORMAL_ALARM: "异常告警",
  WAIT_CONFIRM: "等待确认",
  CONFIRMED: "已确认",
  OBSTACLE_WAIT: "障碍等待",
  REROUTING: "重规划",
  FORBIDDEN_ZONE_WAIT: "禁区等待",
  FINISHED: "任务完成",
  STOPPED: "已停止",
  PATROL: "移动中",
  TAG_DETECTED: "识别中"
};
```

兼容旧状态，但页面主显示使用新版中文。

**Step 2：字段兜底**

读取：

```text
current_shelf
current_target
pose
path
forbidden_zones
shelves
scan
scan.detections
obstacle
alarm
events
```

若字段不存在：

- `current_shelf` 用 `current_zone` 兜底。
- `scan.detected_items` 用空数组。
- `scan.detections` 用空数组。
- `path.waypoints` 用空数组。
- `shelves` 用旧 `zones` 或默认 A1/A2/B1/B2。

**Step 3：待确认事件**

待确认事件统一判断：

```text
event.status === "waiting_confirm"
```

不要再判断中文 `待确认`，否则 2.1 改成英文状态后按钮会失效。

## Task 3.1.3：软件兜底演示

**Files:**

- Modify: `src/inspection_robot/web.py`
- Modify: `src/inspection_robot/templates/dashboard.html`
- Create: `docs/demo_runbook.md`

**Step 1：演示按钮或 demo route**

提供一个无真车也能跑通的流程：

1. 开始巡逻。
2. 规划从起点到 A1/A2/B1/B2 的路径。
3. 模拟到达 A1。
4. 模拟 A1 扫描正常。
5. 模拟前方障碍等待。
6. 模拟障碍解除或重规划。
7. 模拟黑胶带禁区触发和恢复。
8. 模拟 A2 缺失/重复/错放异常。
9. 模拟 AprilTag 与颜色/OCR/图像证据冲突。
10. 人工确认处理。
11. 导出 CSV。

可以先用多个按钮完成，不强求一键自动播放。

**Step 2：不要复制核心规则**

如果 2.1 已实现 `record_scan_result()`，demo route 调这个方法。若尚未实现，demo route 只添加符合契约的事件，标明 `source: "simulate"`。等 2.1 完成后再删掉临时规则。

**Step 3：运行手册**

`docs/demo_runbook.md` 包含：

- 场地布置。
- 打印和粘贴标签：物品标签用 `item_labels_A4.pdf`，货架标签用 `shelf_labels_A4.pdf`；货架标签按上方货架号、下方码摆放。
- 真车演示流程。
- 软件兜底流程。
- 硬件失败时如何讲解。
- 网络失败时如何用终端日志和录屏兜底。
- 答辩 2-3 分钟讲解顺序。

**Verify:**

```bash
scripts/run_local.sh
```

Expected: 没有小车时，队友也能在 2 分钟内演示路径、货架、异常和确认闭环。
