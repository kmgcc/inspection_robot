# 2.1 队友计划：巡检轮次、识别证据与异常规则

**前置依据：** `docs/REAL_REQUIREMENTS.md`、`docs/api_contract.md`、`config/shelf_manifest.json`、`config/tag_map.json`  
**目标：** 实现货架清单、巡检轮次、第一轮跳过缺货、第二轮开始异常判断、多模态证据合并、动态拓扑和事件持久化。

## 基本原则

2.1 是核心规则层，不直接控制电机。它接收 1.1 上报的观察，包括货架、物品、颜色、文字、图形、黑胶带列端、障碍和音频/灯光结果，然后更新状态、拓扑和事件。

旧的固定地图 A* 路径规划不再是真车主链路。若代码中保留 planner，只作为软件兜底或未来扩展。

规则：

1. AprilTag 是稳定主身份。
2. 文字、图形、颜色作为补充证据。
3. 颜色可选，缺失不报错。
4. 证据冲突生成 `evidence_mismatch`，进入人工复核。

## Task 2.1.3a：视觉处理状态机与算法选型

推荐采用“传感器触发 + 视觉确认 + OCR 读取 + 语音反馈”的状态机，不把 OCR、目标检测和语音播报做成常驻重任务。

建议状态：

```text
IDLE
SEARCHING
ALIGNING
SLOW_DOWN_OR_STOP
CAPTURE
RECTIFY
OCR
VERIFY
TTS
DONE
```

职责划分：

1. 传感器触发进入搜索或避障。
2. AprilTag 提供身份、角度、四角位置和稳定性。
3. 轻量目标检测只判断 `card`、`shelf_slot`、`unknown_object` 等少数类别。
4. OpenCV 负责 ROI、卡片轮廓、透视矫正和颜色补充。
5. PaddleOCR 轻量中文模型只处理矫正后的局部文字区域。
6. Piper 或固定音频负责播报，使用队列避免互相打断。

第一阶段可以不训练模型，先用 AprilTag + 轮廓 + OCR。第二阶段再把轻量检测模型放到 AI HAT+ 上低频运行。

## Task 2.1.4：异常判断

从第二轮开始判断：

1. `missing_item`：预期存在但当前轮未识别到。
2. `duplicate_item`：同一货架同一物品重复。
3. `wrong_shelf`：物品出现在错误货架。
4. `unknown_item`：识别到未配置物品。
5. `evidence_mismatch`：识别证据冲突。

缺货事件要求：

1. 事件状态为 `waiting_confirm`。
2. 触发红色灯光提示。
3. 触发语音警报。
4. 网页可人工确认。
