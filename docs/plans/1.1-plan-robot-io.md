# 1.1 队友计划：小车硬件、麦轮运动与侧向感知

**负责人：** 队友 1  
**前置依赖：** `docs/api_contract.md` 和 0.1 基准配置格式  
**目标：** 把真实 RASPBOT V2 小车接入系统，让它能低速执行固定地图路线，使用麦克纳姆轮完成前进、横移和原地旋转，用超声波处理前方障碍，用底部四路红外识别黑胶带禁区边界，用侧向摄像头识别货架和物品标签。

本计划只管硬件输入输出和 runtime 接线，不负责货架异常规则，也不负责网页展示。1.1 上报的是观察：当前位置、标签、障碍、禁区触发、扫描开始/结束。至于这些观察意味着正常、缺失、重复还是错放，由 2.1 判断。

**Step 2：运动原语**

`motion.py` 暴露：

```python
def move_forward_slow() -> None: ...
def move_backward_slow() -> None: ...
def strafe_left_slow() -> None: ...
def strafe_right_slow() -> None: ...
def rotate_left_slow() -> None: ...
def rotate_right_slow() -> None: ...

```

实现时复用 `move_forward`、`move_backward`、`move_left`、`move_right`、`rotate_left`、`rotate_right`。速度先取保守值，例如 20-40，答辩稳定优先。

**Step 3：传感器原语**

`sensors.py` 暴露：

```python
def read_distance_mm() -> int | None: ...
def read_tape_boundary() -> tuple[int, int, int, int] | None: ...
```

超声波读不到时返回 `None`；黑胶带传感器只作为禁区兜底，不作为主巡线控制。四路红外返回值要保留每一路状态，便于判断是左侧、右侧还是前侧先压到黑线；运行策略是先停、后退，再交给 2.1 或 runtime 选择远离货架的一侧避让。

**Step 4：告警原语**

```python

def clear_alarm() -> None: ...
```

## Task 1.1.3：侧向视觉标签输入

**Files:**

- Modify: `src/inspection_robot/vision/tag_detector.py`
- Create: `scripts/test_side_camera_tag_on_car.py`
- Modify: `requirements.txt`

**Step 1：统一视觉观察接口**

`tag_detector.py` 暴露：

```python
from collections.abc import Iterator

def iter_tag_ids(device: int = 0, cooldown_seconds: float = 1.5) -> Iterator[str]:
    """Yield stable shelf/item tag ids from the side camera."""
```

优先识别 AprilTag TAG36H11；如果二维码路线更稳，也可以加 QR provider，但输出统一为字符串 `tag_id`。另外建议提供一个完整观察接口：

```python
def iter_detections(device: int = 0, cooldown_seconds: float = 1.5) -> Iterator[dict[str, object]]:
    """Yield tag/OCR/color/image evidence from the side camera."""
```

货架标签按“上方货架号、下方 AprilTag”识别：AprilTag 是主身份，OCR 读到的 `A1`、`B2` 是复核字段。物品标签按“颜色块、图像、文字、AprilTag”识别：AprilTag 是主身份，颜色、OCR 和图像分类作为证据随同上报。

**Step 2：多帧投票**

移动中识别容易抖。识别到疑似标签后连续读 3-5 帧，取出现次数最多的 ID；同一标签在 `cooldown_seconds` 内不要重复上报。

**Step 3：车上测试脚本**

`scripts/test_side_camera_tag_on_car.py` 只做一件事：打印侧向摄像头看到的标签 ID。

**Verify on car:**

```bash
python3 scripts/test_side_camera_tag_on_car.py
```

Expected: 侧向摄像头对准货架标签时，能稳定打印货架 ID 和 OCR 货架号；对准物品标签时，能打印 AprilTag ID，并至少打印颜色或文字中的一类补充证据。

上电后分别读取白地和黑胶带数值，记录到测试脚本输出中。第一版可以手工设阈值，但要写清白地读数、黑胶带读数和实际阈值。

四路红外不要只合并成一个布尔值。至少保留：

```text

```

**Step 3：禁区动作原则**

检测到黑胶带时：

1. 立即 `stop()`。
2. 后退 5-10 cm。
3. 上报 `record_forbidden_zone(..., blocked=True)` 或临时用 `record_robot_status("FORBIDDEN_ZONE_WAIT", ...)`。
4. 由地图和 `safe_side` 决定左转、右转或横移，不固定机械地永远转 90 度。

## Task 1.1.4：障碍等待与规则式绕行

**Files:**

- Modify: `src/inspection_robot/robot/sensors.py`
- Modify: `src/inspection_robot/robot/motion.py`
- Create: `scripts/test_obstacle_avoidance_on_car.py`

**Step 1：障碍阈值**

建议先使用两个阈值：

```text
blocked_distance_mm = 200
clear_distance_mm = 280
```

连续 3 次小于 blocked 阈值才进入障碍等待，避免超声波抖动。

**Step 2：等待逻辑**

检测到障碍：

1. `stop()`
2. `show_obstacle_wait()`
3. 等待 1-3 秒
4. 再读距离

若障碍解除，恢复原路径；若障碍仍在，执行固定绕行动作或交给 2.1 重新规划。

**Step 3：远离货架优先**

运行时要接收当前货架的 `safe_side` 或等价参数。若摄像头侧面朝向货架，绕行优先向货架相反侧横移，再前进，再横移回路径附近。没有把握时宁可停住告警，不硬挤。黑胶带禁区触发时也遵守同一原则，但优先级高于普通障碍。

**Verify on car:**

```bash
python3 scripts/test_obstacle_avoidance_on_car.py
```

Expected:

- 前方放纸盒，小车停车并告警。
- 拿走纸盒，小车恢复。
- 纸盒不拿走，小车执行保守绕行或报告需要重规划。

## Task 1.1.5：路径点执行与货架对齐

**Files:**

- Create: `src/inspection_robot/runtime.py`
- Create: `scripts/test_waypoint_runtime_on_car.py`

**Step 1：路径点执行**

runtime 从 2.1 提供的路径点读取目标。第一版可以不做连续坐标控制，只把相邻格转换成短动作：

```text
(x+1, y) -> move_forward_slow()
(x-1, y) -> move_backward_slow()
(x, y+1) -> strafe_right_slow()
(x, y-1) -> strafe_left_slow()
heading change -> rotate_left_slow() / rotate_right_slow()
```

每个动作时间短，动作后停车，给视觉和超声波留时间。

**Step 2：货架扫描姿态**

到达 `A1_SCAN` 这类目标点后：

1. 停车。
2. 按地图要求旋转到摄像头面向货架。
3. 开始标签采集。
4. 采集完成后上报 `shelf_scanned` 或调用 `record_scan_result()`。

**Step 3：接入 Store**

在完整方法实现前，可先调用：

```python
store.record_robot_status("MOVING", "正在前往 A1。")
store.record_tag(tag_id, source="camera")
store.record_obstacle(distance_mm, blocked)
```

2.1 完成后再替换为：

```python
store.record_pose(...)
store.record_shelf_arrival(...)
store.record_scan_result(...)
store.record_forbidden_zone(...)
```

**Verify on car:**

```bash
python3 scripts/test_waypoint_runtime_on_car.py
```

Expected: 小车能按短路径移动到一个货架点，调整姿态，打印或上报识别到的货架/物品标签。

## Done Gate

1.1 完成后必须满足：

- `docs/hardware_smoke_matrix.md` 每项有明确结果。
- 小车能前进、后退、横移、旋转、停止。
- 侧向摄像头能稳定输出至少 4 个标签 ID。
- 侧向摄像头能对货架标签输出 AprilTag ID，并能用 OCR 读出至少 2 个货架号。
- 侧向摄像头能对物品标签输出 AprilTag ID，并能识别至少 2 种颜色或 2 类简化图像。
- 超声波能触发停车等待和恢复。
- 黑胶带/巡线传感器可作为禁区兜底读取，并保留四路方向信息。
- runtime 能把真实标签和障碍事件送入 `/api/status` 或事件列表。
- 不再把黑线跟随作为主任务，只把相关资料作为禁区/边界兜底参考。
