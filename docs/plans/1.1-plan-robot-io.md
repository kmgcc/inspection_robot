# 1.1 队友计划：小车硬件、运动、云台与现场提示

**前置依据：** `docs/REAL_REQUIREMENTS.md`、`docs/api_contract.md`  
**目标：** 接管真实 RASPBOT V2 小车，让网页和 runtime 能可靠控制底盘、云台、超声波、四路黑胶带传感器、RGB、蜂鸣器和音频输出。

## 基本原则

1. 不实现循线。
2. 不把黑线当路线。
3. 四路全黑是列端触发，主动作是顺时针原地转 90 度。
4. 非预期禁区或局部压黑按障碍/禁区处理。
5. 小车正常巡逻应保持慢速正向行驶。
6. 运动、声音、灯光和云台必须能被网页手动控制，用于现场调试。

## Task 1.1.2：运动原语

**Files:**

- Modify: `src/inspection_robot/robot/motion.py`

提供：

```python
def move_forward_slow(speed: int = 30, duration_seconds: float = 0.35) -> None: ...
def move_backward_slow(speed: int = 30, duration_seconds: float = 0.35) -> None: ...
def strafe_left_slow(speed: int = 30, duration_seconds: float = 0.35) -> None: ...
def strafe_right_slow(speed: int = 30, duration_seconds: float = 0.35) -> None: ...
def rotate_left_slow(speed: int = 30, duration_seconds: float = 0.55) -> None: ...
def rotate_right_slow(speed: int = 30, duration_seconds: float = 0.55) -> None: ...
def stop() -> None: ...
```

`duration_seconds` 需要现场标定，目标是 90 度转向和一个车身距离可复现。

## Task 1.1.4：四路黑胶带传感器

**Files:**

- Modify: `src/inspection_robot/robot/sensors.py`
- Create/Modify: `scripts/test_tape_boundary_on_car.py`

必须提供：

```python
def read_tape_boundary() -> tuple[int, int, int, int] | None: ...
def full_tape_boundary_detected(state: tuple[int, int, int, int] | None) -> bool: ...
```

语义：

1. `full_tape_boundary_detected` 为真时，表示四路全黑，进入列端转向。
2. 单路或局部黑线不等同于正常转向点。
3. 局部黑线可以作为非预期禁区或安全兜底。

Expected: 小车压到列端横向黑胶带时能稳定识别四路全黑。

## Task 1.1.5：超声波与 6 秒避障

**Files:**

- Modify: `src/inspection_robot/robot/sensors.py`
- Modify: `src/inspection_robot/runtime.py`

逻辑：

1. 检测到前方障碍。
2. 停车。
3. 等待 6 秒。
4. 再次检测。
5. 障碍解除则恢复前进。
6. 障碍仍在则执行绕行动作。

右侧绕行：

1. 顺时针 90 度。
2. 前进一个车身距离。
3. 逆时针 90 度。
4. 前进一个车身距离。
5. 逆时针 90 度。
6. 前进一个车身距离。
7. 顺时针 90 度恢复方向。

绕行必须可被新障碍打断，并支持有限嵌套。

## Task 1.1.6：音频、蜂鸣器与 RGB

**Files:**

- Modify/Create: `src/inspection_robot/robot/alarm.py`
- Modify/Create: audio helper module if needed
- Use: `src/inspection_robot/static/audio/obstacle.wav`
- Use: `src/inspection_robot/static/audio/first.wav`
- Use: `src/inspection_robot/static/audio/following.wav`

要求：

1. 检测到障碍物或非预期禁区播放 `obstacle.wav`。
2. 扫描到货架播放 `first.wav`。
3. 识别到货架上的物品播放 `following.wav`，每检测到一个物品播放一次。
4. 缺货或高优先级异常亮红灯并语音报警；第一版可复用固定音频，后续再接 Piper TTS。
5. 没有音频文件时不影响主流程。
6. 音频失败应记录事件或状态。
7. 音频播放必须非阻塞，不得卡住巡逻、避障、扫描或网页请求。
8. 音频输出使用树莓派默认音频设备，优先调用 `paplay`、`pw-play`、`aplay` 或 `ffplay`。
9. 连续物品识别时要有队列或节流策略，避免音效重叠。

## Done Gate

1. 小车能从网页启动并慢速前进。
2. 手动控制可用。
3. 急停可靠。
4. 云台侧向初始化可用。
5. 四路全黑触发顺时针 90 度。
6. 障碍等待 6 秒后能恢复或绕行。
7. RGB、蜂鸣器或音频提示可被触发。
8. 官方 App 不再占用硬件。
