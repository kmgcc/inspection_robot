# 1.1 Plan Implementation Plan

**前置状态：** 0.1 共享契约已完成，后续开发以 docs/api_contract.md 为准。

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the robot-side input/output layer: AprilTag input, line-following, obstacle wait/resume, buzzer/RGB alarms, and a runtime loop that forwards observations to the shared inspection core.

**Architecture:** This plan owns only hardware-facing adapters and the car runtime. It does not decide business rules; it sends raw observations into `docs/api_contract.md` and the behavior later expanded by `2.1 Plan`. Hardware smoke tests come first, then adapters, then the runtime loop.

**Tech Stack:** Python 3 on Raspberry Pi, RASPBOT V2 official Python examples, optional ROS2 Humble AprilTag fallback, `dt_apriltags`, OpenCV, shell scripts.

---

## 串行关系判断

本计划内部必须按顺序推进：先确认硬件和官方例程可用，再写适配层，再接运行循环。否则很容易把“设备不可用”误判为“业务代码错误”。

跨计划依赖已经满足：最终 runtime wiring 使用 `docs/api_contract.md`，并调用这些方法：

```python
store.record_tag(tag_id: str, observed_zone: str | None = None, source: str = "camera") -> None
store.record_obstacle(distance_mm: int | None, blocked: bool) -> None
store.record_robot_status(status: str, message: str | None = None) -> None
```

本计划可以直接按 `docs/api_contract.md` 做硬件验证和 runtime 接线。`2.1 Plan` 的完整异常规则不阻塞本计划的硬件推进。

## Task 1.1.3: AprilTag 输入适配

**Files:**

- Create: `src/inspection_robot/vision/apriltag_detector.py`
- Create: `src/inspection_robot/vision/ros_bridge.py`
- Create: `scripts/test_tag_on_car.py`
- Modify: `requirements.txt`

**Step 1: 实现 direct camera provider**

在 `apriltag_detector.py` 暴露：

```python
from collections.abc import Iterator


def iter_tag_ids(device: int = 0, cooldown_seconds: float = 1.5) -> Iterator[str]:
    """Yield stable TAG36H11 AprilTag IDs as strings."""
```

实现要求：

- 使用 `cv2.VideoCapture(device)`。
- 使用 `dt_apriltags.Detector(families="tag36h11")`。
- 同一 tag 在 `cooldown_seconds` 内不要重复输出。
- 疑似识别后连续读 3 帧，取出现次数最多的 ID。

**Step 2: 实现 ROS2 fallback provider**

在 `ros_bridge.py` 暴露同名或等价 API：

```python
def iter_tag_ids() -> Iterator[str]:
    """Yield AprilTag IDs from the official ROS2 AprilTag topic."""
```

要求：

- 只在车上 ROS2 环境启用。
- 本地 import 不应失败；缺 ROS2 时给出清晰错误。

**Step 3: 写车上测试脚本**

`scripts/test_tag_on_car.py`：

```python

for index, tag_id in enumerate(iter_tag_ids()):
    print(tag_id, flush=True)
    if index >= 3:
        break
```

如果 direct camera 不通，再临时切到 `ros_bridge.iter_tag_ids`。

**Step 4: Verify on car**


Expected: 能稳定打印至少 4 个标签 ID，终端不会被同一标签刷屏。
